★ 상속(Inheritance)
	- 슈퍼(상위, 부모) 클래스가 가지고 있는 특성을 물려주면
	  서브(하위, 자식) 클래스에서 특성을 물려받는 객체지향 프로그램의 개념이다.
	- 특성 : 상태정보(멤버변수, 필드, 속성), 행위정보(멤버메소드)
	
	- 일반적인 의미의 상속은 통상 메소드를 물려주는 것을 의미하며, 
	  멤버변수의 상속은 잘 사용하지는 않는다. (아예안쓰는건 아니지만)
	◆- extends 키워드를 활용한다 : public class A extends B ▶ A(하위) → B(상위)
		
	- 상속받은 서브(하위)클래스에서 자신만의 상태정보, 행위정보를 추가할 수 있다 : 알파상태
	
	- 상속받은 서브클래스에서 메소드를 수정(재정의)할 수 있다 : Override(오버라이드)
	
	◆- Override(재정의) 필수 조건
	  # 반드시 상속이 이루어진 상태에서 서브클래스에서 구현한다.
	  # 메소드의 이름이 같아야 한다.
	  # 매개변수의 개수와 타입 및 순서가 무조건 똑같아야 한다.
	  # 리턴문이 있을 경우 리턴타입이 같아야 한다.
	- 자바는 단일상속만 허용 한다 : 2개 이상의 클래스를 상속받을 수 없다.
	  이러한 단점을 해결하기 위해 인터페이스(Interface)를 제공(등장)
